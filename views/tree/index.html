• 使用

    <ui-tree>
    </ui-tree>

  • 属性

    属性名 默认值 是否必须 描述
    url 是 获取树的数据
    on-before-click 否 判断是否可以点击, 回调函数名称, 要挂载在顶层controller里面
    on-click 否 点击时触发, 回调函数名称, 要挂载在顶层controller里面
    on-before-check 否 判断是否可以选中, 回调函数名称, 要挂载在顶层controller里面
    on-check 否 选中时触发, 回调函数名称, 要挂载在顶层controller里面
    search-model 否 如果设置该属性，那么会使用$scope.$watch(该属性), 实时过滤tree
    on-add false 否 当移动到树节点的时候会显示一个添加按钮
    on-edit false 否 当移动到树节点的时候会显示一个编辑按钮
    on-del false 否 当移动到树节点的时候会显示一个删除按钮

  • 方法

    方法名 描述
    addTreeNodeBtn(label[标题], className[class名称], handler[点击后回调]) 添加一个当移动到数节点上面的额外按钮
    addTreeNodeBtnAtIndex(index[位置],label[标题], className[class名称], handler[点击后回调]) 添加一个当移动到数节点上面的额外按钮到指定的位置
    setData(data) 手动设置树的数据
    expand(level[级别]) 按级别展开树
    expandById(id) 按id展开树
    selectItems 获取树选中的数据集合
    selectValues 获取树选中的数据ID集合
    dataList 当然树的数据集合
    dataMap 当前树的key-value集合，key为ID
    getDataById() 获取数据
    getHierarchyDataById(id) 获取数据的承继关系
    getById(id) 获取节点
    getHierarchyById(id) 获取节点的承继关系
    filter(text) 过滤显示树

  • 事件

    事件名 参数 触发
    uitree.dataSuccess (evt, dataList) 当数据请求成功后，可以对数据进行二次加工
